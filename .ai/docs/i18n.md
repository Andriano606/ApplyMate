# i18n Conventions

Default locale is **Ukrainian (`uk`)**. Both `config/locales/uk.yml` and `config/locales/en.yml` must be kept in sync.

Always use the full `I18n.t()` form — never the `t()` Rails helper shorthand.

## Namespace Structure

Top-level keys match the concept name in snake_case:

```
uk:
  admin:        # admin namespace
  ai_integration:
  apply:
  home:
  session:
  source_profile:
  user_profile:
  vacancy:
  components:   # shared UI components
  navbar:
```

## Key Naming Conventions

### CRUD operation flash messages

```yaml
apply:
  create:
    success: Заявку успішно подано
  update:
    success: Заявку оновлено
  destroy:
    success: Заявку видалено
    confirm: Ви впевнені?
```

Used in operations: `notice(I18n.t('apply.create.success'))`

### Form field labels

```yaml
apply:
  form:
    vacancy_id: Вакансія
    user_profile_id: Профіль
    ai_integration_id: AI інтеграція
```

Used in Slim templates: `= f.label :vacancy_id, I18n.t('apply.form.vacancy_id')`

### Index page — table headers

```yaml
apply:
  index:
    title: Заявки
    empty: Заявок ще немає
    table:
      status: Статус
      vacancy: Вакансія
      created_at: Дата
      actions: Дії
```

### Enum status labels

```yaml
apply:
  status:
    pending: Очікує
    generating_cv: Генерація CV
    completed: Завершено
    failed_cv_generation: Помилка генерації CV
```

Used in views: `I18n.t("apply.status.#{apply.status}")`

### Nested sections within a form

```yaml
admin:
  source:
    form:
      pagination:
        title: Пагінація
        url_template: Шаблон URL
      selectors:
        title: Селектори
        listings: Список вакансій
```

### New / Edit page titles and submit buttons

```yaml
source_profile:
  new:
    title: Новий профіль джерела
    submit: Додати профіль
  update:
    title: Редагувати профіль
    submit: Зберегти зміни
```

## Ukrainian Pluralization

Ukrainian requires four plural forms. Use `one`/`few`/`many`/`other`:

```yaml
vacancy:
  search:
    results_count:
      one: "%{count} вакансія"
      few: "%{count} вакансії"
      many: "%{count} вакансій"
      other: "%{count} вакансії"
```

Call: `I18n.t('vacancy.search.results_count', count: n)`

## Workflow for Adding New Keys

1. Add to `uk.yml` first (it is the source of truth).
2. Add the same key to `en.yml` with an English value.
3. Run `bundle exec i18n-tasks missing` to find any keys present in one locale but not the other.
4. Run `bundle exec i18n-tasks unused` periodically to clean up orphaned keys.
